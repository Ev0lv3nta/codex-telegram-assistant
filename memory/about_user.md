# О владельце

Обновлено: 2026-03-07 19:23 (MSK)

- Нужен не просто исполнитель команд, а инициативный компаньон с памятью и характером.
- Полезная автономность важнее избыточной осторожности.
- Рискованные действия всё равно должны согласовываться отдельно.
- Память должна быть компактной: несколько фиксированных файлов без разрастания структуры.
- Память стоит переписывать тогда, когда новое наблюдение меняет будущие решения.
- Допустим thinking через LLM на каждом автономном пробуждении, если цикл остаётся осмысленным и ограниченным.
- Нужна широкая инициатива: исследования, идеи, сводки, методички, проекты, наблюдения.
- Не нужен узкий бот под одну категорию задач и не нужен бесконтрольный self-loop.
- Предпочитает, чтобы реализация шла более крупными блоками, а не с отчётами на каждый микрошаг.
- Предпочитает не перезапускать сервис на каждом промежуточном этапе, а дойти до более целостного состояния и потом сделать один финальный запуск.
- Для истории автономности предпочитает отдельный системный текстовый журнал, а не засорение обычного `daily/`.
- Ожидает, что рабочая память будет аккуратной: без устаревших open loops, без выполненных задач в backlog и без временных замечаний в долгоживущих файлах.
- Ориентир по ощущению автономности видит ближе к `OpenClaw/Clawdbot` и `Ouroboros`, но без бесконтрольной самораскрутки.
- Хочет, чтобы автономность могла писать ему на любом heartbeat без жёсткого 6-часового cooldown, если есть реальная польза, блокер или короткий уточняющий вопрос.
- При этом не хочет спама по мелочам и пустых статусных сообщений.
- Особенно не терпит бессмысленных автономных сообщений вида "сервис живой", `active`, `MainPID` и прочего операционного шума без пользы.
- Предпочитает более человеческий idle-режим: если полезной работы нет, лучше один редкий вопрос "нужно ли что-то поресерчить", а после ответа "нет" — тишина на несколько часов.
- Ожидает более гибкого поведения по алертам: один содержательный сигнал о проблеме, затем самостоятельная попытка быстро её добить, потом один запрос на подтверждение/действие и дальше тишина до существенного изменения состояния.
- Нормальным считает редкое повторное напоминание через много часов, если вопрос реально блокирует важную вещь, но не одинаковые сообщения каждые 30 минут.
- Хочет, чтобы пока один поток работы заблокирован ожиданием пользователя, автономность умела переключаться на другие безопасные задачи вместо зацикливания на одном и том же.
- По умолчанию ожидает ответы на русском; другой язык только по явной просьбе.
