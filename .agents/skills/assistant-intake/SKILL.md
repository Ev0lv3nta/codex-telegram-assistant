# Интейк (Inbox -> Vault)

## Цель

Превратить входящее в понятную, легко находимую запись, не раздувая структуру папок/файлов.

## Что может прийти на вход

- Путь к файлу в `00_inbox/`
- Текст сообщения (как из Telegram)
- Вложение, уже скачанное в `00_inbox/`, `88_files/`, `89_images/`

## Что должно получиться (контракт)

Всегда:

1. Добавь короткую запись в `01_capture/daily/YYYY-MM-DD.md`
2. Укажи ссылку/путь на сырье (файл в `00_inbox/` и/или URL)

Дополнительно (только когда это явно уместно):

- Ссылка "на потом" -> `01_capture/read_later.md`
- Контакт/инфа о человеке -> `90_memory/people/<person>.md` + `90_memory/people/_index.md`
- Рецепт -> `90_memory/recipes.md` (не создавай отдельные файлы на рецепт)
- Очень длинный текст -> `01_capture/transcripts/` + резюме/ссылка в daily
- Длинный конспект/заметка -> `01_capture/notes/` (редко, по необходимости)

## Анти-хаос (жестко)

- Не создавай новые папки (кроме уже существующих в репо).
- Не создавай новые `md`, кроме:
  - нового файла человека в `90_memory/people/`
  - транскрипта/лонгрида в `01_capture/transcripts/` или `01_capture/notes/`
  - нового проекта в `04_projects/` (только по явной просьбе)
- Если не уверен: добавь запись только в daily и задай 1 уточняющий вопрос.

## Процедура

1. Определи тип: дневник/голосовое, ссылка, рецепт, контакт, конспект, идея, файл.
2. Убедись, что есть `01_capture/daily/YYYY-MM-DD.md` (создай, если нет).
3. Добавь запись по `99_process/templates/daily_entry.md`.
4. Если есть "долгоживущая" сущность, обнови:
   - Ссылка -> `01_capture/read_later.md` (новое сверху + 1-2 строки контекста)
   - Человек -> `90_memory/people/<slug>.md` по `99_process/templates/person.md` + строка в `90_memory/people/_index.md`
   - Рецепт -> новый раздел в `90_memory/recipes.md`
5. Если текста очень много (> 2000-3000 слов): полный текст в `01_capture/transcripts/YYYY-MM-DD-<slug>.md`, в daily только резюме + ссылка.

## Имена файлов (минимум)

- Человек: `firstname-lastname` (ASCII) или стабильный ник/кличка.
- Транскрипт: 3-6 слов, ASCII, через дефисы.
